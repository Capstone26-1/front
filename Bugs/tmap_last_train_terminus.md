# [버그] Tmap 막차 종착역 미반영 — 실제 도달 불가 경로 반환

## 재현 쿼리

"인덕원역에서 돌곶이역갈건데 23시 40분에 출발할거야"

## 증상

- 추천 경로: 인덕원 → (4호선) → **삼각지** → (6호선) → 돌곶이 (환승 포함)
- 실제 상황: 23:40 출발 기준 인덕원역 4호선 막차(23:44)는 **사당 종착**
  → 사당에서 열차가 끊기므로 삼각지까지 도달 불가

## 원인

Tmap 대중교통 경로 API는 **정기 스케줄 기반**으로 경로를 반환합니다.
막차가 중간 역(사당)에서 종착하는 경우, 이후 구간(사당→삼각지)의 연결 열차가
실제로 존재하지 않더라도 이론상 경로로 반환합니다.

즉, Tmap이 "열차가 사당에서 종착한다"는 사실을 경로 계산에 반영하지 못하는 것으로 추정됩니다.

## 관련 인프라

- `server/tools/tmap.js` → `searchTransitRoute`
- Tmap Transit API: `POST https://apis.openapi.sk.com/transit/routes`

## 영향

위험도 계산이 실제보다 낮게 산출될 수 있음.
막차가 종착역 이전에서 끊기는 경로를 "가능한 경로"로 표시해 사용자를 혼란시킴.

## 해결 방향 (보류)

1. **단기**: 시스템 프롬프트에 "23:00 이후 환승 포함 경로는 실제 막차 종착역을 고려해 보수적으로 판단하라" 규칙 추가
2. **장기**: Tmap 응답의 각 leg 출발 시각과 실제 막차 시각표를 대조 검증하는 로직 구현 (별도 막차 시각표 DB 필요)
