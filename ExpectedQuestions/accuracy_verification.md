# Q. 막차 판단의 정확도는 어떻게 검증했나? 틀린 경우는 없나?

## 핵심 답변

정확도는 두 가지 레이어로 구성됩니다:
실시간 API 데이터의 정확성과 Claude의 추론 정확성입니다.

---

## 레이어 1: 실시간 API 데이터

경로 탐색은 Tmap 대중교통 API를 사용합니다.
Tmap은 서울 대중교통 실시간 운행 정보를 기반으로 경로를 제공하는
국내 최고 수준의 상용 API입니다.

- 경로 존재 여부 → Tmap API 응답 기준 (신뢰도 높음)
- 소요 시간 → 실시간 배차 간격 반영
- 막차 종료 여부 → 경로 없음으로 판단

**한계**: Tmap API가 반환하는 막차 시간이 실제 운행과 1~3분 오차가 있을 수 있습니다.

---

## 레이어 2: Claude의 추론

Claude는 API 데이터를 받아 아래를 계산합니다:
- 현재 시각 vs 막차 도착 예정 시각 → 여유 시간 계산
- 여유 시간 기준으로 안전/주의/위험/매우 위험 분류
- MCP 툴 결과(날씨, 도로, 지하철)를 종합해 최종 판정

**잠재적 오류 케이스**:

| 상황 | 위험 |
|---|---|
| 환승이 많은 복잡한 경로 | 환승 대기 시간 과소 추정 가능 |
| 막차 직전 1~2분 여유 | 경계 케이스에서 판정 불안정 |
| 임시 운행 변경 (명절 등) | Tmap API 반영 지연 가능 |

---

## 검증 방법

시나리오 A~D를 직접 실행하여 tool 호출 흐름과 최종 판정을 확인했습니다.

- 시나리오 A (평상시): 안전 판정, tool 2개 호출 확인
- 시나리오 C (날씨): weather_alert_tool 자동 호출 확인
- 실제 Tmap 경로 데이터와 판정 결과 육안 검증 수행
