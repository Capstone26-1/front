# Q. API 비용 문제는 없나? 실제 서비스라면 Anthropic API 호출 비용이 부담되지 않나?

## 핵심 답변

현재는 학술 목적의 데모이므로 비용 최적화보다 기능 완성도를 우선했습니다.
실제 서비스화 시에는 아래와 같은 방법으로 비용을 크게 줄일 수 있습니다.

---

## 현재 구성

- 모델: `claude-opus-4-8` (최고 성능, 가장 비싼 모델)
- 호출 방식: 매 쿼리마다 최대 6회 반복 (tool_use 루프)
- 최적화: 미적용

---

## 비용 절감 방법

### 1. 모델 다운그레이드
```
claude-opus-4-8  → 고비용, 최고 추론 품질  (현재)
claude-sonnet-4-6 → 중간 비용, 충분한 품질  (권장)
claude-haiku-4-5  → 저비용, 단순 쿼리에 적합
```
단순한 막차 판단에는 Sonnet으로도 충분합니다.

### 2. 프롬프트 캐싱 (Prompt Caching)
시스템 프롬프트는 매 호출마다 동일합니다.
Anthropic의 Prompt Caching을 적용하면 반복 토큰 비용을 약 90% 절감할 수 있습니다.

```javascript
system: [{
  type: "text",
  text: getSystemPrompt(),
  cache_control: { type: "ephemeral" }  // 캐시 적용
}]
```

### 3. 툴 호출 횟수 제한
현재 최대 6회 루프 → 실용적으로는 3~4회면 충분.

---

## 실제 서비스 비용 추산 (Sonnet 기준, 캐싱 적용 시)

| 항목 | 수치 |
|---|---|
| 쿼리당 평균 토큰 | ~3,000 tokens |
| 캐싱 적용 후 비용 | 약 $0.003 / 쿼리 |
| 월 1만 쿼리 기준 | 약 $30 / 월 |
