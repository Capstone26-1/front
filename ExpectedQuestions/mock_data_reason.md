# Q. public_event_tool은 mock 데이터인데, 실제 데이터를 못 쓴 이유가 뭔가?

## 핵심 답변

국내에 공개된 실시간 대형 행사 API가 존재하지 않습니다.
mock 데이터는 의도적 설계이며, 시연의 한계가 아닙니다.

---

## 왜 실제 API가 없는가

한국의 공공 데이터 포털(data.go.kr, data.seoul.go.kr)을 조사한 결과:

| 데이터 | API 존재 여부 |
|---|---|
| 기상청 날씨 | ✅ data.go.kr |
| 국토부 도로 돌발상황 | ✅ openapi.its.go.kr |
| 서울 지하철 실시간 | ✅ data.seoul.go.kr |
| **대형 행사 실시간 정보** | ❌ 공개 API 없음 |

야구·축구 경기 일정은 각 구단/리그 공식 사이트에서 제공하지만,
**실시간 종료 여부·관중 수를 API로 제공하는 공공 서비스가 없습니다.**

---

## Mock 데이터의 역할

`public_event_tool`의 목적은 데이터 자체가 아니라
**"특정 상황에서 Agent가 이 툴을 동적으로 선택한다"는 것을 시연하는 것**입니다.

```javascript
// 잠실 → 야구 경기 mock
잠실: { eventName: "LG vs KT 야구 경기", estimatedCrowd: 25000, ... }
상암: { eventName: "K리그 경기", estimatedCrowd: 40000, ... }
```

실제 서비스에서는 네이버 스포츠 API, 각 구단 공식 API,
또는 크롤링으로 대체할 수 있습니다.

---

## 한 줄 요약

> "실시간 행사 데이터의 공개 API가 국내에 존재하지 않아 mock을 사용했으며,
> 툴의 동적 선택 동작 자체는 실제 API와 동일하게 작동합니다."
