# Q. 뉴스 API가 실시간이 아닌데 신뢰할 수 있나요?

## 답변

네이버 뉴스 검색 API는 최신 기사부터 정렬(`sort=date`)해서 가져옵니다.
사고·통제 같은 교통 이슈는 발생 직후 기사화되는 특성이 있어,
실시간 API 대비 수십 분~수 시간의 지연은 있지만 당일 내 대부분 감지 가능합니다.

## 핵심 포인트

저희 시스템에서 뉴스는 역할이 명확히 분리되어 있습니다:

| 역할 | 담당 API |
|---|---|
| 경로 판단 (실시간) | Tmap, 서울 지하철 API, ITS API |
| 원인 파악 (Why) | 네이버 뉴스 API |

뉴스는 "막혔다"를 판단하는 게 아니라 "왜 막혔는지"를 설명하는 데 사용됩니다.
따라서 수십 분의 지연이 있어도 원인 파악 목적에는 충분합니다.

## 보완 가능성

- 뉴스 기반 이슈가 감지되면 → 해당 영역의 실시간 API(ITS, 지하철)를 추가 호출
- 즉, 뉴스는 "단서"를 제공하고 실시간 API가 "확인"하는 2단계 구조
