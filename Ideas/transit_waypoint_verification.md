# [아이디어] 환승역 막차 검증 알고리즘

## 문제 배경

Tmap 대중교통 경로 API는 **스케줄 기반**으로 경로를 반환합니다.
심야 시간대 막차가 중간 역(예: 사당)에서 종착하는 경우에도,
이론상 이후 구간(사당→삼각지→돌곶이)까지 가는 경로를 반환합니다.

**재현 예시**
- 쿼리: "인덕원역에서 돌곶이역 23시 40분 출발"
- Tmap 반환 경로: 인덕원 → (4호선) → 삼각지 → (6호선) → 돌곶이
- 실제 상황: 4호선 23:44 인덕원 출발 막차는 **사당 종착** → 삼각지 도달 불가

## 해결 알고리즘

### 핵심 아이디어
경유지(환승역)마다 서울 지하철 실시간 API를 호출하고,
**Anthropic Claude API**가 각 구간의 실현 가능성을 노선 지식 기반으로 검증합니다.

### 데이터 흐름

```
1. Tmap → 경로 반환 (인덕원 → 삼각지 → 돌곶이)

2. 출발역 조회: transit_disruption_tool("인덕원")
   → lastTrainDestination: "사당"

3. 환승역 조회: transit_disruption_tool("삼각지")
   → trains: [] (현재 운행 없음) or 운행 중

4. Claude 추론:
   - 인덕원 막차 행선지: "사당"
   - 삼각지는 4호선에서 사당 이북(사당→동작→이촌→삼각지 순)
   - → 사당 종착 열차로는 삼각지 도달 불가
   - → riskScore 100, 매우 위험

5. taxi_fare_tool 호출:
   - 마지막 도달 가능 역(사당) → 목적지(돌곶이) 구간 택시 요금 계산
```

### 구현 변경 사항

| 파일 | 변경 내용 |
|------|---------|
| `server/mcpServer.js` | `transitDisruptionHandler`에 `lastTrainDestination` 필드 추가 |
| `server/claude.js` | 시스템 프롬프트에 환승역 검증 규칙 추가 |

### 활용 포인트

- 서울 지하철 실시간 API의 `lstcarAt`, `trainLineNm` 필드 활용
- Claude의 서울 지하철 노선 지식(역 순서, 방향)으로 도달 가능성 추론
- Anthropic API 비용이 충분하므로 경유지마다 검증 호출 가능
